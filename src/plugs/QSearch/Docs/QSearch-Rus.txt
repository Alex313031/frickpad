---------------------------------------------------------------
QSearch - плагин для текстового редактора AkelPad
---------------------------------------------------------------

Плагин представляет собой панель для "быстрого" поиска текста.
Предполагается, что для вызова функции плагина (QSearch::QSearch)
будет назначена горячая клавиша (например, F7).

Клавиши панели быстрого поиска:
Enter, F3        - искать следующий (вниз)
Shift+Enter      - искать предыдущий (вверх)
Alt+Enter        - искать с начала файла
Alt+Shift+Enter  - найти все
Ctrl+Shift+Enter - найти все (во всех файлах)
Ctrl+Enter       - подхватить выделенный текст
Esc              - скрыть панель
Ctrl+Del         - строка поиска: стереть текст после (правее) каретки
Ctrl+BS          - строка поиска: стереть текст до (левее) каретки
Ctrl+A           - строка поиска: выделить весь текст
Смотрите также описание 'use_alt_hotkeys'.
Примечание: кнопка "Найти все" имеет собственное всплывающее меню.

Функции FindNext, FindPrev и FindAll позволяют назначить "горячие клавиши"
для вызова из редактора, а также использовать их возможности при вызове из
плагина Hotkeys или из скриптов (с помощью плагина Scripts).
Рассмотрим их подробнее.

FindNext - искать следующий (вниз).
Рекомендуемая "горячая клавиша" - F3.
Эта функция поддерживает автозагрузку для скрытой загрузки плагина QSearch
при запуске AkelPad. (Цель этого - доступность всех возможностей плагина
сразу при старте AkelPad, а не после первого показа панели поиска QSearch).
Дополнительные возможности на примере плагина Hotkeys:
 -------------------------------------------------------------
 Имя              |  Команда                       |  Клавиша
 -------------------------------------------------------------
 Quick Find Next  |  Call("QSearch::FindNext", 1)  |  Ctrl+F3
 -------------------------------------------------------------
Назначает Ctrl+F3 на действие "подхватить выделенный текст и искать
далее (вниз)".
При отсутствии выделенного текста, выделяется текущее слово (согласно
настройке 'select_by_fnd').
При обеих установленных галочках "Search: Pick up selected text" и
"Window: Catch main F3", функция FindNext подхватывает выделенный текст или
выделяет текущее слово согласно настройке 'select_by_f3'.

FindPrev - искать предыдущий (вверх).
Рекомендуемая "горячая клавиша" - Shift+F3.
Дополнительные возможности на примере плагина Hotkeys:
 -------------------------------------------------------------------
 Имя              |  Команда                       |  Клавиша
 -------------------------------------------------------------------
 Quick Find Prev  |  Call("QSearch::FindPrev", 1)  |  Ctrl+Shift+F3
 -------------------------------------------------------------------
Назначает Ctrl+Shift+F3 на действие "подхватить выделенный текст и
искать далее (вверх)".
При отсутствии выделенного текста, выделяется текущее слово (согласно
настройке 'select_by_fnd').
При обеих установленных галочках "Search: Pick up selected text" и
"Window: Catch main F3", функция FindPrev подхватывает выделенный текст или
выделяет текущее слово согласно настройке 'select_by_f3'.

FindAll - найти все.
Рекомендуется использовать через плагин Hotkeys.
Дополнительные возможности на примере плагина Hotkeys:
 -------------------------------------------------------------------------
 Имя                 |  Команда                      |  Клавиша
 -------------------------------------------------------------------------
 Quick Find All (1)  |  Call("QSearch::FindAll", 1)  |  Ctrl+Alt+F3
 Quick Find All (2)  |  Call("QSearch::FindAll", 2)  |  Alt+Shift+F3
 Quick Find All (3)  |  Call("QSearch::FindAll", 3)  |  Ctrl+Alt+Shift+F3
 -------------------------------------------------------------------------
(1) Назначает Ctrl+Alt+F3 на действие "подхватить выделенный текст и
найти все в текущем документе".
При отсутствии выделенного текста, выделяется текущее слово (согласно
настройке 'select_by_fnd').
(2) Назначает Alt+Shift+F3 на действие "найти все во всех документах".
(3) Назначает Ctrl+Alt+Shift+F3 на действие "подхватить выделенный текст и
найти все во всех документах".
При отсутствии выделенного текста, выделяется текущее слово (согласно
настройке 'select_by_fnd').
При обеих установленных галочках "Search: Pick up selected text" и
"Window: Catch main F3", функция FindAll подхватывает выделенный текст или
выделяет текущее слово согласно настройке 'select_by_f3'.

Также плагин предоставляет две функции: SelFindNext и SelFindPrev -
для быстрого поиска выделенного слова/фразы ниже или выше по тексту
прямо в окне редактирования.
При отсутствии выделенного текста, выделяется текущее слово (согласно
настройке 'select_by_selfnd').
Поведение этих функций зависит от общих настроек поиска, заданных в
панели QSearch (или в ini-файле, см. ниже).
При этом функции QSearch::QSearch и QSearch::SelFindNext/Prev могут
искать разный текст (QSearch::QSearch - текст в строке поиска, а
QSearch::SelFindNext/Prev - текст, выделенный в окне редактирования).
Предполагается, что для вызова этих функций будут назначены горячие
клавиши (например, Ctrl+Alt+Right и Ctrl+Alt+Left).
Пример использования: выделяем слово в окне редактирования и вызываем
SelFindNext или SelFindPrev. В результате выделение переходит к
следующему или предыдущему слову, совпадающему с выделенным изначально.

Функция DialogSwitcher позволяет переключаться между диалогами "Поиск",
"Замена", "Перейти к строке" и панелью QSearch с помощью "горячих клавиш"
этих диалогов. Например, если нажать Ctrl+H в диалоге "Поиск", он
изменится на диалог "Замена". При использовании функции DialogSwitcher
рекомендуется поставить галочку для её автоматической загрузки в окне
настроек плагинов.

Совет: тихий мгновенный поиск по F3.
Тихий мгновенный поиск позволяет искать выделенное слово (фразу) простым
нажатием F3 или Shift+F3 прямо в окне редактирования. Поиск "тихий" потому,
что не требует открытую панель QSearch для своей работы (так же, как и
"тихие" функции QSearch::SelFindNext и QSearch::SelFindPrev).
Для активации этой возможности выполните следующие три шага:
1. Включите "Поиск: SelFind подхватывает выделенный текст" во всплывающем
   меню панели QSearch.
   Это позволяет QSearch подхватить текст, выделенный в окне редактирования.
2. Включите "Окно: Обрабатывать F3 от главного окна" во всплывающем меню
   панели QSearch.
   Это позволяет QSearch обработать F3/Shift+F3, нажатые в окне редактирования.
3. Поставьте галочку "QSearch::SelFindNext" в окне Настройки -> Плагины...
   Это позволяет скрыто загрузить QSearch при запуске AkelPad.


Дополнительные настройки (всплывающее меню панели QSearch или файл
"AkelFiles\Plugs\QSearch.ini"):

1) "Search: On-the-fly mode"
   "Поиск: Искать на лету"
   srch_onthefly_mode = 1
     Поиск производится непосредственно во время набора текста в
     строке поиска QSearch.
   srch_onthefly_mode = 0
     Поиск производится только по нажатию Enter.

2) "Search: Always from beginning"
   "Поиск: Всегда с начала"
   srch_from_beginning = 0
     При изменении слова в строке поиска он (поиск) будет продолжен
     с текущей позиции в файле.
   srch_from_beginning = 1
     Введенный текст будет искаться с начала файла.

3) "Search: Use special characters ( \n, \t, * and ? )"
   "Поиск: Использовать спец. символы ( \n, \t, * и ? )"
   srch_use_specialchars = 1
     В строке поиска могут использоваться спец. символы:
       \n - перевод строки
       \t - табуляция
       \\ - обратный слэш ( '\' )
       \* - звёздочка  ( '*' )
       \? - знак вопроса ( '?' )
       ?  - любой символ ( например, 'b', '0', ' ' и т.д. )
       *  - 0 или больше любых символов (т.e. любая (под)строка)
       ** - 0 или больше любых непробельных символов ("Слово целиком" вкл.)
     Примечание:
       ? и * не работают с переводом строки, \n должен быть указан явно.
   srch_use_specialchars = 0
     Обычный поиск, без спец. символов.

4) "Search: Use regular expressions"
   "Поиск: Использовать регулярные выражения"
   srch_use_regexp = 0
     Обычный поиск, без регулярных выражений.
   srch_use_regexp = 1
     Поиск с регулярными выражениями. Используется реализация регулярных
     выражений в AkelPad.

5) "Search: . matches \n in reg.expressions"
   "Поиск: . захватывает \n в рег.выражениях"
   srch_regexp_dot_newline = 0
     При поиске с регулярными выражениями,  .  не захватывает  \n
   srch_regexp_dot_newline = 1
     При поиске с регулярными выражениями,  .  захватывает  \n

6) "Search: Pick up selected text"
   "Поиск: Подхватывать выделенный текст"
   srch_pickup_selection = 1
     При вызове QSearch или при нажатии F3 в окне редактирования AkelPad,
     в строку поиска QSearch вставляется текст, выделенный в окне
     редактирования AkelPad. То же самое при нажатии кнопок Find Next
     и Find Prev в панели QSearch.
   srch_pickup_selection = 0
     При вызове QSearch или при нажатии F3 в окне редактирования AkelPad,
     текст в строке поиска не изменяется.
   srch_pickup_selection = 10
     То же, что srch_pickup_selection = 0.
     При установке галочки "Search: Pick up selected text" значение параметра
     srch_pickup_selection устанавливается в 11.
   srch_pickup_selection = 11
     При вызове QSearch или при нажатии F3 в окне редактирования AkelPad,
     в строку поиска QSearch вставляется текст, выделенный в окне
     редактирования AkelPad. При нажатии же кнопок Find Next и Find Prev
     (в панели QSearch) всегда ищется текст из строки поиска QSearch - т.е.
     выделенный в окне редактирования AkelPad текст не подхватывается.
     При снятии галочки "Search: Pick up selected text" значение параметра
     srch_pickup_selection устанавливается в 10.

7) "Search: SelFind picks up selected text"
   "Поиск: SelFind подхватывает выделенный текст"
   srch_selfind_pickup = 0
     Вызов SelFindNext или SelFindPrev не меняет текст в строке поиска
     панели QSearch.
   srch_selfind_pickup = 1
     Вызов SelFindNext или SelFindPrev вставляет в строку поиска панели
     QSearch текст, выделенный в окне редактирования AkelPad.

8) "Search: Stop at EOF (end of file)"
   "Поиск: Останавливаться в конце файла"
   srch_stop_eof = 0
     При достижении конца файла поиск продолжается с начала этого файла.
   srch_stop_eof = 1
     При достижении конца файла показывается соответствующее сообщение.
   srch_stop_eof = 10
     То же, что srch_stop_eof = 0.
     При установке галочки "Search: Stop at EOF" значение параметра
     srch_stop_eof устанавливается в 11.
   srch_stop_eof = 11
     При достижении конца файла поиск прекращается. Молча. Без показа
     сообщения.
     При снятии галочки "Search: Stop at EOF" значение параметра
     srch_stop_eof устанавливается в 10.

9) "Window: Docked top"
   "Окно: Расположить сверху"
   srch_wnd_dockedtop = 1
     Панель QSearch находится в верхней клиентской части окна AkelPad.
   srch_wnd_dockedtop = 0
     Панель QSearch находится в нижней клиентской части окна AkelPad.

10) "Window: Select all when focused"
   "Окно: Фокус ввода выделяет всё"
   edit_focus_selectall = 1
     Каждый раз, когда строка поиска QSearch получает фокус ввода,
     весь текст в строке поиска становится выделенным.
   edit_focus_selectall = 0
     Весь текст в строке поиска выделяется только при явном вызове
     QSearch::QSearch.

11) "Window: Catch main F3"
   "Окно: Обрабатывать F3 от главного окна"
   catch_main_f3 = 1
     Нажатие клавиши F3 (Shift+F3) в окне редактора AkelPad продолжает
     поиск плагином QSearch, если до этого не вызывалось стандартное
     окно поиска или замены AkelPad.
   catch_main_f3 = 0
     Стандартное поведение AkelPad.

12) "Window: Catch main Esc"
   "Окно: Обрабатывать Esc от главного окна"
   catch_main_esc = 1
     Нажатие клавиши Esc в окне редактора AkelPad скрывает панель QSearch,
     если она была видима.
   catch_main_esc = 0
     Панель QSearch не перехватывает клавишу Esc; она обрабатывается самим
     AkelPad (по умолчанию).
   Также можно вручную установить одно из следующих значений:
   catch_main_esc = 10
     Нажатие клавиши Esc в панели QSearch передаётся на обработку самому
     редактору AkelPad (по умолчанию - закрывает программу).
     При установке галочки "Window: Catch main Esc" значение параметра
     catch_main_esc устанавливается в 11.
   catch_main_esc = 11
     Работает так же, как при catch_main_esc = 1.
     При снятии галочки "Window: Catch main Esc" значение параметра
     catch_main_esc устанавливается в 10.

13) "Window: Hotkey hides the panel"
   "Окно: Hotkey скрывает панель"
   hotkey_hides_panel = 1
     При нажатии "горячей клавиши" плагина в панели QSearch, она (панель)
     скрывается (аналог нажатия Esc).
     Повторный вызов функции QSearch::QSearch также закрывает панель.
     (Равно как и нажатие "горячей клавиши" плагина в окне редактирования
     AkelPad).
   hotkey_hides_panel = 0
     При нажатии "горячей клавиши" плагина в панели QSearch, фокус ввода
     передается окну редактирования AkelPad.
   hotkey_hides_panel = 11
     При нажатии "горячей клавиши" плагина в панели QSearch, она (панель)
     скрывается (аналог нажатия Esc).
     При нажатии "горячей клавиши" плагина в окне редактирования AkelPad,
     панель QSearch становится активной.
   hotkey_hides_panel = 10
     Работает так же, как при hotkey_hides_panel = 0.
     При снятии галочки "Window: Hotkey hides the panel" значение параметра
     'hotkey_hides_panel' устанавливается в 11.

14) "Window: Editor auto-focus (on mouse move)"
   "Окно: Авто-фокус в редактор (движением мыши)"
   editor_autofocus = 1
     При qsearch_autofocus = 1:
       Если панель QSearch имеет фокус ввода и указатель мыши из строки
       поиска панели QSearch передвинулся в окно редактирования AkelPad,
       то фокус ввода переходит к окну редактирования AkelPad.
       Если указатель мыши передвинулся в строку поиска панели QSearch, то
       фокус ввода переходит к строке поиска.
     При qsearch_autofocus = 0:
       Окно редактирования AkelPad всегда получает фокус ввода при движении
       указателя мыши в этом окне (даже если указатель мыши при этом не
       покидал строку поиска QSearch).
       Если указатель мыши передвинулся в строку поиска панели QSearch, то
       фокус ввода переходит к строке поиска.
   editor_autofocus = 0
     Ничего особенного не происходит: окно редактирования AkelPad не получает
     фокус ввода автоматически при наведении указателя мыши.

15) "Window: QSearch auto-focus (on mouse move)"
   "Окно: Авто-фокус в QSearch (движением мыши)"
   qsearch_autofocus = 1
     Если указатель мыши передвинулся в строку поиска панели QSearch, то
     фокус ввода переходит к строке поиска.
   qsearch_autofocus = 0
     При editor_autofocus = 0:
       Ничего особенного не происходит: строка поиска QSearch не получает
       фокус ввода при наведении на неё указателя мыши.
     При editor_autofocus = 1:
       Если указатель мыши передвинулся в строку поиска панели QSearch, то
       фокус ввода переходит к строке поиска.

16) "Window: QSearch auto-focus (on file open)"
   "Окно: Авто-фокус в QSearch (при откр. файла)"
   qsearch_autofocus_file = 1
     При открытии файла и переключении на другой файл (документ), фокус ввода
     переходит к строке поиска панели QSearch.
   qsearch_autofocus_file = 0
     Ничего особенного не происходит: QSearch не перехватывает фокус ввода.

17) dock_rect_disabled = 1
     Если значение этого параметра вручную установлено в 1, то координаты
     "встроенной" панели QSearch не будут сохраняться.

18) dock_rect = ...
     (внутренний параметр, сохраняется автоматически при 'dock_rect_disabled'
     равном 0)

19) color_notfound = FFE0E000
     Цвет фона, который устанавливается в случае, когда искомый текст не
     найден. Этот цвет задается в виде "RRGGBB00", где RR, GG и BB - это
     шестнадцатеричные значения красного, зеленого и синего цветов (от 00
     до FF).

20) color_notregexp=F8E0F800
     Цвет фона, который устанавливается в случае неудачного регулярного
     выражения. Этот цвет задается в виде "RRGGBB00", где RR, GG и BB -
     это шестнадцатеричные значения красного, зеленого и синего цветов
     (от 00 до FF).

21) color_eof = E4FFE400
     Цвет фона, который устанавливается при пересечении конца файла (когда
     поиск продолжается с начала файла). Этот цвет задается в виде
     "RRGGBB00", где RR, GG и BB - это шестнадцатеричные значения красного,
     зеленого и синего цветов (от 00 до FF).
     При установке "color_eof = FFFFFF00" цвет меняться не будет.

22) color_highlight = C0FFC000
     Цвет фона для текста, который подсвечивается при установленной галочке
     "Highlight all". Этот цвет задается в виде "RRGGBB00", где RR, GG и BB -
     это шестнадцатеричные значения красного, зеленого и синего цветов (от 00
     до FF).

23) highlight_mark_id = 1001
     Параметр 'ID' для плагина HighLight когда QSearch вызывает его функцию
     "Coder::HighLight" для подсветки (пометки) текста.

24) highlight_state = 0
    (сохраняется автоматически при вкл/выкл галочки "Highlight all").
    Также можно вручную установить одно из следующих значений:
    highlight_state = 10
      При запуске AkelPad галочка "Highlight all" всегда не установлена.
    highlight_state = 11
      При запуске AkelPad галочка "Highlight all" всегда установлена.

25) use_alt_hotkeys = 1
     В панели QSearch доступны следующие дополнительные сочетания клавиш:
     Alt+alt_match_case    (Alt+C) - Учитывать регистр (case) вкл/выкл
     Alt+alt_whole_word    (Alt+W) - Слово целиком (whole word) вкл/выкл
     Alt+alt_search_mode   (Alt+R) - Режим поиска (текст/спец./рег.выражения)
     Alt+alt_highlight_all (Alt+H) - Подсветить все (highlight) вкл/выкл
    use_alt_hotkeys = 0
     Дополнительные сочетания клавиш в панели QSearch отключены.

26) alt_match_case = 67
     Если 'use_alt_hotkeys' включено (установлено в 1), этот параметр задаёт
     код быстрой клавиши для галочки "Учитывать регистр" (панель QSearch).
     Значение 67 соответствует клавише 'C', т.е. получаем сочетание Alt+C.
     Код клавиши может быть посчитан как (англ.буква - 'A' + 65).
     То есть 65 соответствует Alt+A, 66 - Alt+B, 67 - Alt+C и т.д.
     Также, 48 соответствует Alt+0, 49 - Alt+1, 50 - Alt+2 и т.д.
     Для детальной информации по кодам клавиш см. страницу Virtual-Key Codes
     в MSDN.

27) alt_whole_word = 87
     Если 'use_alt_hotkeys' включено (установлено в 1), этот параметр задаёт
     код быстрой клавиши для галочки "Слово целиком" (панель QSearch).
     Значение 87 соответствует клавише 'W', т.е. получаем сочетание Alt+W.
     См. 'alt_match_case' для более детальной информации.

28) alt_search_mode = 82
     Если 'use_alt_hotkeys' включено (установлено в 1), этот параметр задаёт
     код быстрой клавиши для переключения режима поиска:
     текст -> спец. символы -> регулярные выражения -> текст -> ...
     Значение 82 соответствует клавише 'R', т.е. получаем сочетание Alt+R.
     См. 'alt_match_case' для более детальной информации.

29) alt_highlight_all = 72
     Если 'use_alt_hotkeys' включено (установлено в 1), этот параметр задаёт
     код быстрой клавиши для галочки "Подсветить все" (панель QSearch).
     Значение 72 соответствует клавише 'H', т.е. получаем сочетание Alt+H.
     См. 'alt_match_case' для более детальной информации.

30) find_history_items = 15
     Максимальное количество элементов в истории поиска (выпадающем списке).
     Установка этого значения в 0 отключает историю поиска.

31) history_save = 3
     Может быть установлено в 0 или быть суммой следующих значений:
     0 - история поиска и состояние галочек не сохраняется.
     1 - история поиска сохраняется в Системном Реестре.
     2 - состояние галочек "Учитывать регистр" и "Слово целиком"
         сохраняется в Системном Реестре.
     4 - сохранять не в Системном Реестре, а там, куда сохраняет
         настройки AkelPad (например, в .ini файл).
         Позволяет сделать настройки QSearch полностью портабельными.

32) new_ui = 0
     QSearch показывает свой старый "классический" интерфейс:
     [искомый текст]  [ ]Учит.регистр [ ]Сл.целиком [ ]Подсв.все
    new_ui = 1
     QSearch показывает обновлённый "классический" интерфейс + две кнопки:
     [искомый текст]  [ < ] [ > ]  [ ]Учит.регистр [ ]Сл.целиком [ ]Подсв.все
    new_ui = 2
     QSearch показывает обновлённый "классический" интерфейс + три кнопки:
     [искомый текст]  [ < ] [ > ] [ * ]  [ ]Учит.регистр [ ]Сл.целиком [ ]Подсв.все
       - кнопка [ * ] означает "Найти все" и имеет собственное всплывающее меню

33) select_by_f3 = 0
     Актуально при установленных галочках "Search: Pick up selected text" и
     "Window: Catch main F3".
     Когда в окне редактирования нет выделенного текста, F3 использует текст
     из панели QSearch для продолжения поиска.
    select_by_f3 = 1
     Актуально при установленных галочках "Search: Pick up selected text" и
     "Window: Catch main F3".
     Когда в окне редактирования нет выделенного текста, F3 выделяет слово
     возле каретки, чтобы использовать его для дальнейшего поиска.
     (Пример: сл|ово, слово|, |слово.)
    select_by_f3 = 2
     Актуально при установленных галочках "Search: Pick up selected text" и
     "Window: Catch main F3".
     Когда в окне редактирования нет выделенного текста, F3 выделяет слово,
     чтобы использовать его для дальнейшего поиска, если каретка находится
     внутри слова - т.е. не перед первым или не после последнего символа
     этого слова.
     (Пример: с|лово, сл|ово, сло|во, слов|о - но не слово| или |слово.)

34) select_by_fnd = 1
     Эта настройка контролирует, будет ли выделяться текущее слово вызовом
      Call("QSearch::FindNext", 1), Call("QSearch::FindPrev", 1),
      Call("QSearch::FindAll", 1) и Call("QSearch::FindAll", 3)
     при отсутствии выделенного текста в редакторе.
     Значение 1 выделяет слово возле каретки (пример: сл|ово, слово|, |слово).
     Значение 2 выделяет слово, если каретка находится внутри слова - т.е.
     не перед первым или не после последнего символа этого слова (пример:
     с|лово, сл|ово, сло|во, слов|о - но не слово| или |слово).
     Значение 0 не выделяет слово.

35) select_by_selfnd = 1
     Эта настройка контролирует, будет ли выделяться текущее слово вызовом
      QSearch::SelFindNext и QSearch::SelFindPrev
     при отсутствии выделенного текста в редакторе.
     Значение 1 выделяет слово возле каретки (пример: сл|ово, слово|, |слово).
     Значение 2 выделяет слово, если каретка находится внутри слова - т.е.
     не перед первым или не после последнего символа этого слова (пример:
     с|лово, сл|ово, сло|во, слов|о - но не слово| или |слово).
     Значение 0 не выделяет слово.

36) adj_incompl_regexp = 1
     Корректирует "незавершённое" регулярное выражение: символы "+", "*",
     "+?" and "*?" в конце регулярного выражения игнорируются, чтобы
     избежать выделения до конца файла.
     Это особенно полезно в режиме поиска "на лету", когда символ "+" или
     "*" встречается в середине регулярного выражения в процессе набора.
     Чтобы специально использовать эти символы в конце регулярного выражения,
     просто добавьте в его конец дополнительный символ '+' или '*': например,
     печатайте "a++" вместо "a+" и печатайте "a*?+" вместо "a*?".
    adj_incompl_regexp = 2
     Корректирует "незавершённое" регулярное выражение только в режиме
     поиска "на лету".
    adj_incompl_regexp = 0
     Отключает коррекцию "незавершённого" регулярного выражения.

37) findall_mode = 2
     Отвечает за режим "Найти все".
     Младший байт может иметь значение:
      1 - Только подсчёт
      2 - Вывод в Log::Output
      3 - Вывод в файл (N вкладок)
      4 - Вывод в файл (1 вкладка)
     К этому значению может быть прибавлено:
      256 - Авто-подсчёт всех вхождений
     Так, значение findall_mode = 258 будет означать:
      * Авто-подсчёт всех вхождений (во время поиска)
      * Вывод в Log::Output (при нажатии кнопки "Найти все")

38) findall_result = 272
     Соответствует диалогу "Find All Results Format".
     Это значение будет суммой следующих чисел:
      0 - Показывать строку со вхождением
      1 - Показывать только само совпадение (вхождение)
      2 - Режим фильтра: чистый текст без заголовка/результата/позиции
      16 - Показывать позицию: (Стр:Поз)
      32 - Показывать длину совпадения: (Длина)
      256 - Показывать заголовок: "Ищем %s ..."
      4096 - Применить Coder alias (синтаксическую тему)

39) findall_count_delay = 400
     Задержка, в миллисекундах, перед подсчётом всех вхождений, когда обе
     настройки "Авто-подсчёт всех вхождений" и "Искать на лету" активны.

40) logoutput_frp_mode = 3
     Режим отображения результатов поиска (FindResultsPolicy) при выводе в
     панель Log::Output.
     Определяет значение (смысл) параметров 'frp_before' и 'frp_after'.
     Может принимать одно из следующих значений:
      1 (QSFRM_LINE)       - количество строк
      2 (QSFRM_LINE_CR)    - кол-во строк + завершающий перенос строки
      3 (QSFRM_CHARINLINE) - кол-во символов внутри текущей строки
      4 (QSFRM_CHAR)       - количество символов

41) logoutput_frp_before = 100
     В зависимости от 'logoutput_frp_mode', задаёт число строк или символов,
     которые будут показаны _перед_ каждым вхождением при выводе в панель
     Log::Output.
      При logoutput_frp_mode = 1, это число строк перед строкой со вхождением.
     И logoutput_frp_before = 0 будет означать: "использовать всю строку, в
     которой найдено вхождение".
      При logoutput_frp_mode = 2, это число строк перед строкой со вхождением,
     плюс после каждого выведенного в панель Log::Output вхождения будет
     добавлен перенос строки.
      При logoutput_frp_mode = 3, это число символов перед вхождением,
     ограниченное строкой со вхождением (т.е. символы вне этой строки не
     захватываются). И logoutput_frp_before = 0 будет означать:
     "использовать только само совпадение".
      При logoutput_frp_mode = 4, это число символов перед вхождением, не
     ограниченное строкой со вхождением (т.е. могут захватываться символы из
     предыдущих строк). И logoutput_frp_before = 0 будет означать:
     "использовать только само совпадение".

42) logoutput_frp_after = 100
     В зависимости от 'logoutput_frp_mode', задаёт число строк или символов,
     которые будут показаны _после_ каждого вхождения при выводе в панель
     Log::Output.
     См. описание 'logoutput_frp_before'.

43) logoutput_frp_highlight = 1
     Отвечает за подсветку найденных совпадений в результатах поиска.
     Может принимать одно из следующих значений:
      0 (QSFRH_NONE)      - не подсвечивать
      1 (QSFRH_SELECT)    - выделить искомый текст

44) fileoutput_frp_mode = 1
     Режим отображения результатов поиска (FindResultsPolicy) при выводе в
     файл.
     См. описание 'logoutput_frp_mode'.

45) fileoutput_frp_before = 0
     В зависимости от 'fileoutput_frp_mode', задаёт число строк или символов,
     которые будут показаны _перед_ каждым вхождением при выводе в файл.
     См. описание 'logoutput_frp_before'.

46) fileoutput_frp_after = 0
     В зависимости от 'fileoutput_frp_mode', задаёт число строк или символов,
     которые будут показаны _после_ каждого вхождения при выводе в файл.
     См. описание 'logoutput_frp_before'.

47) fileoutput_frp_highlight = 2
     Отвечает за подсветку найденных совпадений в результатах поиска.
     Может принимать одно из следующих значений:
      0 (QSFRH_NONE)      - не подсвечивать
      1 (QSFRH_SELECT)    - выделить искомый текст
      2 (QSFRH_IFCHECKED) - подсвечивать, если стоит галочка "Подсветить все"
      3 (QSFRH_ALWAYS)    - подсвечивать всегда

48) edit_minwidth = 0
     Минимальная ширина строки поиска панели QSearch.
     Значение 0 означает: не меньше, чем стандартная ширина, использовавшаяся
     в предыдущих версиях QSearch (когда панель не поддерживала изменение
     размеров).

49) edit_maxwidth = 640
     Максимальная ширина строки поиска панели QSearch.
     Значение 0 означает: без ограничений (т.е. ширина строки поиска всегда
     пропорциональна текущей ширине панели QSearch).
